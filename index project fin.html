Backend — Overview

Backend development refers to the server-side part of a web application. It is responsible for processing data, handling requests from the frontend, and communicating with databases.

While the frontend is what users see in the browser, the backend works behind the scenes to make everything function properly.

Responsibilities of Backend Development

  • Handle user requests
  • Process application logic
  • Store and retrieve data
  • Manage authentication and security
  • Connect frontend with databases

Popular Backend Languages

  • Python
  • JavaScript (Node.js)
  • Java
  • PHP
  • Ruby
  • C#

Backend — Servers & Runtimes

Servers
A server is a computer or software that provides services to other computers (clients). When a user visits a website, the browser sends a request to a server. The server processes the request and sends back a response.

Request Flow (example)

  1. User opens a website
  2. Browser sends request to server
  3. Server processes the request
  4. Server sends webpage back to the browser

Popular Web Servers

  • Apache
  • Nginx
  • Node.js Server
  • Microsoft IIS

Runtimes

A runtime environment is a platform that allows programs to run on a server.

RuntimeLanguage
Node.jsJavaScript
Python RuntimePython
Java Virtual Machine (JVM)Java

Example: Node.js allows JavaScript to run on the server instead of only in the browser.

Backend — Databases

A database is used to store and manage data for a website or application (for example: user accounts, passwords, product details, orders, messages).

Types of Databases

TypeExample
Relational DatabaseMySQL, PostgreSQL
NoSQL DatabaseMongoDB, Firebase

Example Table (Users)

Backend — APIs & Authentication

APIs (Application Programming Interfaces) allow different software systems to communicate with each other. The frontend sends requests to backend APIs, the backend processes the request, and returns data (usually JSON) to the frontend.

Example API Request

GET /users

Example Response (JSON)

[
  { "name": "Harini" },
  { "name": "Madhu" }
]

Authentication

Authentication is the process of verifying the identity of a user (login with username/password, OTP, token-based methods).

Common Authentication Methods:

  • Password Authentication
  • JWT (JSON Web Token)
  • OAuth
  • Session Authentication

Authentication helps keep user data secure.

Java — Overview

Java is a high-level, object-oriented programming language widely used for building web applications, mobile applications, and enterprise software.

Java was developed by James Gosling at Sun Microsystems in 1995.

Java follows the principle: "Write Once, Run Anywhere (WORA)" — Java programs can run on any system that has a Java Virtual Machine (JVM).

Features of Java

  • Object-Oriented Programming (OOP)
  • Platform Independent
  • Secure
  • Multithreaded
  • Robust and reliable
  • Large community support

Uses of Java

  • Web applications
  • Android mobile apps
  • Desktop software
  • Enterprise systems
  • Cloud applications

Java is based on OOP concepts which help organize code into objects and classes.

Main OOP Concepts

  • Object — Instance of a class
  • Inheritance — One class inherits another
  • Encapsulation — Data hiding
  • Polymorphism — Same method with different behavior
  • Abstraction — Hiding implementation details

Java — JVM & Tooling

JVM (Java Virtual Machine) is responsible for running Java programs.

How a Java Program Runs

  1. Write Java code
  2. Compile using Java Compiler (javac)
  3. Convert to bytecode (.class file)
  4. JVM executes the bytecode

This allows Java to run on any operating system with a compatible JVM.

It was developed by Bjarne Stroustrup in 1985 as an extension of the C programming language.

C++ supports both procedural programming and object-oriented programming (OOP).

Features of C++

  • Portable across platforms
  • Large standard library

Uses of C++

  • Game development
  • Operating systems
  • Embedded systems
  • Desktop applications
  • High-performance software

C++ — STL & Libraries

The STL (Standard Template Library) is a powerful C++ library that provides ready-made data structures and algorithms.

Components of STL

  • Containers — vector, list, map
  • Algorithms — sort, search
  • Iterators — traverse containers

Example — Vector

#include <iostream>
#include <vector>
using namespace std;

int main(){

    vector numbers = {1,2,3,4,5};

    for(int n : numbers){
        cout << n << " ";
    }

    return 0;
}

C++ — Memory & Pointers

C++ allows direct memory management using pointers.

Pointer Concept

A pointer is a variable that stores the memory address of another variable.

#include <iostream>
using namespace std;

int main(){

    int a = 10;

    int *ptr = &a;

    cout << a << endl;
    cout << ptr << endl;

    return 0;
}

Explanation

SymbolMeaning
&Address of variable
*Pointer variable

Dynamic Memory Allocation

#include <iostream>
using namespace std;

int main(){

    int *num = new int;

    *num = 50;

    cout << *num;

    delete num;

    return 0;
}
